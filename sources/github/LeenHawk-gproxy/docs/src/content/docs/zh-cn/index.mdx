## 为什么选择 GPROXY?

<CardGrid stagger>
	<Card title="多供应商接入" icon="puzzle">
		OpenAI、Anthropic、Gemini / Vertex、DeepSeek、Groq、OpenRouter、NVIDIA、
		Claude Code、Codex 等等 —— 统一通过 OpenAI / Claude / Gemini 兼容接口暴露。
	</Card>
	<Card title="同协议透传" icon="rocket">
		当客户端与上游使用相同协议时，GPROXY 以最小解析开销直接透传字节 ——
		低延迟、高吞吐。
	</Card>
	<Card title="原生多租户" icon="seti:lock">
		用户、API 密钥、模型权限、按模型的限流、成本配额 —— 都是一等公民，
		而不是事后补丁。
	</Card>
	<Card title="内嵌控制台" icon="laptop">
		React 控制台直接编译进二进制，挂载在 <code>/console</code>，
		无需单独部署前端。
	</Card>
	<Card title="可插拔存储" icon="seti:db">
		默认使用 SQLite，也支持 PostgreSQL 和 MySQL（通过 SeaORM / SQLx）。
		可选的 XChaCha20-Poly1305 磁盘加密。
	</Card>
	<Card title="Rust SDK" icon="seti:rust">
		<code>gproxy-sdk</code> 再导出了协议、路由、供应商三个 crate，
		方便你把引擎嵌入到自己的服务中。
	</Card>
</CardGrid>
