## Why GPROXY?

<CardGrid stagger>
	<Card title="Multi-provider" icon="puzzle">
		OpenAI, Anthropic, Gemini / Vertex, DeepSeek, Groq, OpenRouter, NVIDIA, Claude Code,
		Codex, and more — behind a unified OpenAI / Claude / Gemini compatible surface.
	</Card>
	<Card title="Same-protocol passthrough" icon="rocket">
		When the client and upstream speak the same protocol, GPROXY forwards bytes with
		minimal parsing — low overhead, high throughput.
	</Card>
	<Card title="Multi-tenant by design" icon="seti:lock">
		Users, API keys, model permissions, per-model rate limits, and cost quotas are
		first-class primitives, not afterthoughts.
	</Card>
	<Card title="Embedded console" icon="laptop">
		A React console is compiled into the binary and served at <code>/console</code>.
		No separate frontend to deploy.
	</Card>
	<Card title="Pluggable storage" icon="seti:db">
		SQLite out of the box, PostgreSQL and MySQL via SeaORM / SQLx. Optional
		at-rest encryption with XChaCha20-Poly1305.
	</Card>
	<Card title="Rust SDK" icon="seti:rust">
		<code>gproxy-sdk</code> re-exports the protocol, routing, and provider
		crates so you can embed the engine into your own service.
	</Card>
</CardGrid>
